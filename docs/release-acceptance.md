# Release Acceptance (Windows x64)

## Acceptance Environment

Release acceptance must be executed on a **fresh** machine with one of:

- Windows 10 x64
- Windows 11 x64

Environment constraints:

- `adb` is **not preinstalled**
- `scrcpy` is **not preinstalled**
- Validation can be completed **offline** (no dependency on `winget` online install)

## Minimum Acceptance Items

The following items are the minimum go/no-go criteria:

- [ ] Double-clicking `WiFiPhoneControl.exe` starts the application successfully.
- [ ] Pairing workflow can be entered (UI reaches pairing steps and accepts pairing inputs).
- [ ] `Start Control` can launch the `scrcpy` process (assuming device connectivity prerequisites are met).
- [ ] No dependency on `winget` automatic install during acceptance.

## Release Artifact Gate

Before publishing any release ZIP, complete both sections below:

### 1) Artifact Check

- [ ] Release artifact name is exactly `WiFiPhoneControl-vX.Y.Z-win-x64.zip`
- [ ] Artifact is generated by `package-release.cmd` and stored under `release\`
- [ ] ZIP contains `WiFiPhoneControl.exe`
- [ ] ZIP contains `core/`, `config/`, and `profiles/`
- [ ] ZIP contains bundled runtime dependencies under `runtime/` (including `scrcpy` and `adb`)

### 2) Manual Acceptance Sign-off

- [ ] Acceptance environment matches this document
- [ ] All minimum acceptance items are checked
- [ ] Tester and date are recorded in release notes

> If any item above is not checked, the release **must not** publish `WiFiPhoneControl-vX.Y.Z-win-x64.zip`.

## Release Note Requirements

Each release note must explicitly include:

1. **System Requirements**
   - Windows 10/11 x64
2. **Bundled Dependency Versions**
   - `scrcpy` version
   - `adb` version
3. **Verification Statement**
   - Confirmed in fresh/offline acceptance environment
